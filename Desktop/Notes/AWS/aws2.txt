EC2
====
EC2 stands for Elastic Cloud Compute.

EC2 comes in IaaS (Infrastructure As a Service)

It is a webservice provided by the AWS to create servers(instances) in cloud.


Linux provide two ways authentication 

1) Using username and password (recommanded)

2) Using username with key (not recommanded)




Steps to create Linux Server using username and password
========================================================
step1:
------
	Login to AWS Account.
	ex:
		Root User:
			gmail :  ........
			password : ........
step2:
------
	Select the region.
	ex:
		N Virginia 	

step3:
------
	Goto EC2 and create a security group.	
	ex:
		mySecurityGroupBatch55
step4:
------
	Create a EC2 Instance.
	ex:
		AMI : DevOps-LabImage-Centos-8	
		username : centos 
		password : DevOps321
	
step5:
-----
	Download and install putty in your computer.
	ex:
		https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html


step6:
-----
	Copy public IP address and paste in putty to connect with linux server.
	ex:
		login as : centos 
		password : DevOps321

step7:
------
	Terminate the EC2 instance.

Launch Template
================
Launch template is used to create EC2 instance only by just one click.

Launch template contains configuration information of EC2 instance.

It contains ID of an AMI, Instance type (t3.micro), Security Group , key and value pair and other parameters required to launch EC2 instance.

ex:
	AMI 		: DevOps-LabImage-Centos-8
	Region 		: N Virginia
	username 	: centos
	password	: DevOps321 

step1:
------
	Login to AWS account.

step2:
------
	Goto EC2 service and create a launch template.

step3:
------
	Launch EC2 instance (spot instance) from launch template.

step4:
------
	Copy public IP address and paste in putty.

step5:
------
	We need to cancel the request to terminate spot instances.
