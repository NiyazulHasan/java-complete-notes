Views
======
A view is a virtual representation of a data from one or more than one table.

A table which is used to create a view is called base table or above table.

View does not consume memory.

View does not have data.

View gets the data when we run select command.

We have following list of views.

1) Simple view 

2) Complex view 

3) With read only view 

4) With check option view 

5) Materialized view 


1) Simple view
---------------
If a view is created by using one base table is called simple view.

In simple view DML operations are allowed.

ex:
---
	create view v1 as select * from student;

	select * from v1;

	delete from v1 where sno=105;

	select * from v1;

	select * from student;


2) Complex view 
---------------
If a view is created by using more than one base table is called complex view.

In complex view DML operations are not allowed.

ex:
	create view v2 as select e.eid,e.ename,e.esal,d.dname,d.dloc 
				from emp e,dept d where(e.deptno=d.deptno);

	select * from v2;

	delete from v2 where eid=201; --cannot delete from view


3) With read only view 
----------------------
If a view is created by using one base table and DML operations are not required then we need to use with read only view.

ex:
	create view v3 as select * from emp with read only;

	select * from v3;

	delete from v3 where eid=206; -- cannot delete from view


4) With check option view 
---------------------------
If a view is created by using one base table and DML operations are allowed only when condition is satisfied then we need to use with check option view.

ex:
	create view v4 as select * from emp where deptno=30 with check option;

	select * from v4;

	insert into v4 values(207,'Lisa',18000,30,'Salesman',500);

	insert into v4 values(207,'Lisa',18000,50,'Salesman',500); -- invalid 
	
	select * from v4;
	
	select * from emp;


5) Materialized view 
---------------------
To create a materized view a table must have primary key.

A materialized view is also known as SNAPSHOT.

ex:
	alter table emp ADD primary key(eid);

	create materialized view v5 as select * from emp;

	select * from v5; -- 7 records 
	select * from emp; -- 7 records 
	
	delete from emp where eid=207;
	commit;
	
	select * from emp; -- 6 records 
	select * from v5; -- 7 records 

In order to get the changes in materialized view then we need to refresh materized view.
ex:
	exec  DBMS_SNAPSHOT.REFRESH('V5');	

	select * from v5; -- 6 records 


Q) Write a query to see the list of views present in database?

	select view_name from user_views;


Q) Write a query to drop the view?

	drop view v1;
	drop view v2;
	drop view v3;
	drop view v4;
	drop materialized view v5;

Merge Command 
=============
A merge command is a combination of insert and update command.

student10 table 
---------------
drop table student10;
create table student10(sno number(3),sname varchar2(10),sadd varchar2(12));
insert into student10 values(101,'raja','hyd');
insert into student10 values(102,'ravi','delhi');
insert into student10 values(103,'ramana','vizag');
commit;

student20 table 
---------------
drop table student20;
create table student20(sno number(3),sname varchar2(10),sadd varchar2(12));
insert into student20 values(103,'Alan','Florida');
insert into student20 values(104,'James','Texas');
commit;

merge into student10 s1 
using student20 s2 
ON(s1.sno=s2.sno)
when matched then 
update set s1.sname=s2.sname,s1.sadd=s2.sadd
when not matched then 
insert values(s2.sno,s2.sname,s2.sadd);


Sub Queries 
===========
Sometimes we will declare a query inside another query such concept is called sub query.

In sub query , first inner query will execute then outer query.

Sub queries are used to select the records based on unknown values.

Sub queries can be nested upto 32 levels.

We have following list of sub queries.

1) Scalar Sub Query 

2) Inline View 

3) Single Row Sub Query 

4) Multiple Row Sub Query

5) Multiple Column Sub Query 


1) Scalar Sub Query
-------------------
A sub query which returns only one value is called scalar sub query.
It contains one row and one column.

Q) Write a query to display average salary of each employee?

	select ename,(select avg(esal) from emp) as AVG_SAL  from emp;



2) Inline View 
-----------------
If a subquery in FROM clause, then it is called inline view.
An inline view:
	> Returns multiple rows
	> Acts like a temporary table
	> Is used in FROM clause
ex:
	
	select * from (select * from emp order by eid desc) where rownum<=3;


3) Single Row Sub Query
-----------------------
If a sub query returns more then only one row is called single row sub query.


Q) Write a query to display second highest salary from employee table?

	select max(esal) from emp where esal<(select max(esal) from emp);


Q) Write a query to display employees information whose salary greater then 
	Kelvin salary?

	select * from emp where esal>(select esal from emp where ename='Kelvin');	

4) Multiple Row Sub Query
--------------------------
If a sub query returns more then one row is called multiple row sub query.

To perform multiple row sub query we required multiple row operators.

We have three multiple row operators.

1) ALL 

2) ANY 

3) IN 

ex:
---
	select * from emp where esal > ALL (select esal from emp where deptno=10);

	select * from emp where esal > ANY (select esal from emp where deptno=10);
	
	select * from emp where esal IN (select esal from emp where deptno=10);

5) Multiple Column Sub Query 
---------------------------
If a sub query returns more then one column is called muliple column sub query.

ex:
 select * from emp where(eid,ename,esal) IN (select eid,ename,esal from emp);
 or
 select eid,ename,esal from emp where(eid,ename,esal) IN (select eid,ename,esal from emp);
