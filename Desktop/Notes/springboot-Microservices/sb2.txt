Q) Explain @SpringBootApplication annotation?

It is used to mark the main class of a spring boot application.

It is a combination of three annotations.

1) @EnableAutoConfiguration 
----------------------------
	It is used to enable auto configuration mechanism in spring boot.

2) @ComponentScan
----------------
	It is used to scan on packages where our project is located.

3) @Configuration 
-----------------
	It is used to add extra beans to context.
	It is used to perform extra configurations.




Spring Boot Starter
===================
It is one of the component of spring boot.

It is used to simplify dependency management by collecting common dependencies in a single starter.

It contains many built-in starters to develop applications easily and rapidly.

It is also known as dependency descriptor.

All built-in starters follow below naming pattern.
ex:
	spring-boot-starter-*

	Here '*' name of the application

	spring-boot-starter-webmvc
	spring-boot-starter-data-jpa 
	spring-boot-starter-webmvc-test
	spring-boot-starter-validation 
	and etc.


Spring Web Dependency 
=====================
Spring Web is a core dependency in Spring / Spring Boot that is used to build web applications and RESTful APIs.

Spring Web helps your Java application handle HTTP requests and responses.

It contains Spring MVC, REST and Tomcat as embedded server.

We need to add below dependency to develop web applications inside pom.xml file.
ex:
	<dependency>
		<groupId>org.springframework.boot</groupId>
		<artifactId>spring-boot-starter-webmvc</artifactId>
	</dependency>

There are two important features of spring web dependency.

1) It is compatible for web applications. 
2) It perform Autoconfigurations.

Spring Web dependency performs the following auto-configurations:

DispatcherServlet
---------------------
Automatically configures DispatcherServlet, which acts as the front controller in Spring MVC.It receives all incoming HTTP requests and routes them to the appropriate controller methods.

Error Pages
------------
Provides default error handling for common HTTP errors like 404, 500, 400, etc.
Automatically maps error pages using Spring Boot’s built-in error mechanism.

Static Resource Management
-----------------------------
Automatically configures handling of static resources such as CSS, JavaScript, images, etc.Resources placed in folders like static, public, or resources are served without extra configuration.

Servlet Container
-------------------
Automatically configures an embedded servlet container (Tomcat by default).
No need to install or configure an external web server separately.

Interview Questions 
===================

Q) What is @Controller annotation?

It is a stereotype annotation.
It is a class level annotation.
It serves a class as a controller to intercept all incoming HTTP requests. 



Q) What is @RequestMapping annotation? 

It is a method level annotation.
It is used to map the request to perticular handler method.
It is used for all HTTP methods i.e GET,POST,PUT,DELETE and etc.
ex:
	@RequestMapping(value = "/msg", method = RequestMethod.GET)
	@RequestMapping(value = "/msg", method = RequestMethod.POST)
	@RequestMapping(value = "/msg", method = RequestMethod.PUT)
	@RequestMapping(value = "/msg", method = RequestMethod.DELETE)

	
Q) What is @ResponseBody annotation?

It is a method level annotation.
It tells to the controller a return object automatically serialized to Text,JSON or XML.
