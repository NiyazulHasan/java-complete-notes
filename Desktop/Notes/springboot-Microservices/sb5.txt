@RestController
===============
It is used to create RESTful webservices using @RestController annotation.

It is a class level annotation and makes our class to intercept all incoming requests.

It is a combination of two annotations i.e @Controller and @ResponseBody.

We have following list of REST annotations.
ex:
	HTTP Method		Annotation 
	-----------		----------
	GET			@GetMapping 
	POST			@PostMapping 
	PUT 			@PutMapping 
	DELETE 			@DeleteMapping 
	PATCH 			@PatchMapping 
	and etc.

How to read the data from properties file 
==========================================
In Spring Boot, you can read properties file data using the @Value annotation, the Environment abstraction, or the type-safe @ConfigurationProperties.



Approach1
---------
The @Value annotation in Spring Boot is used to inject values from external sources (such as property files, environment variables, or system properties) into fields, method parameters, or constructor parameters in Spring-managed beans.

package com.ihub.www.controller;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HomeController 
{
	@Value("${spring.application.name}")
	private String name;
	
	@Value("${server.port}")
	private String port;
	
	@GetMapping("/fetch")
	public String getDetails()
	{
		return "Application Name :"+name+" | Port No :"+port;
	}
}

package com.ihub.www.controller;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.core.env.Environment;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HomeController 
{
	//constructor injection 
	private final Environment env;
	public HomeController(Environment env)
	{
		this.env = env;
	}
	
	@GetMapping("/fetch")
	public String getDetails()
	{
		String name = env.getProperty("spring.application.name");
		String port = env.getProperty("server.port");
		return  "Application name : "+name+" | Port No : "+port;
	}
}

package com.ihub.www.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

import com.ihub.www.config.AppConfig;

@RestController
public class HomeController 
{	
	//constructor injection 
	private final AppConfig config;
	
	public HomeController(AppConfig config)
	{
		this.config = config;
	}
	
	@GetMapping("/fetch")
	public String getDetails()
	{
		return config.getName();

Q)what is @Component annotation ?
================================
The @Component annotation in Spring Boot is a class-level annotation that marks a Java class as a Spring-managed bean. This makes the class eligible for automatic detection (via classpath scanning) and dependency injection by the Spring IoC (Inversion of Control) container during application startup. 




Q) What is the difference between Monolithic Architecture vs Microservice Architecture?

Monolithic Architecture 
=======================
It is a software design pattern in which all application components are tightly coupled and build as single unified unit.

For example, In monolithic architecture we develop independent services and at the end of the development we pack all the services into a single war file. Later we deploy that war file on top of the server.

Diagram: sb5.1

Advantages:

1) Simple Develop

2) Simple Deploy 

3) Simple Test 

Disadvantages:

1) Large and Complex application

2) Blocks continue development

3) Slow Development 

4) Inflexible 

5) Inreliable
