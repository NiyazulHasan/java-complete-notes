Python Set 
==========
A Python set is an unordered collection used to store multiple unique items in a single variable.

Python sets are:

Unordered (insertion order is not guaranteed)
Mutable (elements can be added or removed after creation)
Heterogeneous (different data types are allowed)
Do not allow duplicates
Allow None values (but only one None)

ex:
---
You create a set using curly braces {} or the set() constructor.

fruits = {"apple", "banana", "mango"}
print(fruits)

Output:

{'banana', 'apple', 'mango'}

ex:
---
nums = {1, 2, 2, 3, 3, 3}
print(nums)

Output:

{1, 2, 3}


ex:
--
You can add or remove elements.

s = {1, 2}
s.add(3)
print(s)

Output:

{1, 2, 3}

ex:
---
Elements Must Be Immutable

Allowed:

int
float
string
tuple

Not allowed:

list
dictionary
another set

Example:

s = {[1, 2], [3, 4]}

This gives:

TypeError: unhashable type: 'list'

ex:
---
a = set()
a.add(3);
print(a)

ex:

ex:
---
Adds multiple items.

s = {1, 2}
s.update([3, 4, 5])
print(s)

Output:

{1, 2, 3, 4, 5}


ex:
---
Removes an item.

s = {1, 2, 3}
s.remove(2)
print(s)

Output:

{1, 3}


ex:
---
Safer than remove.

s.discard(10)

No error occurs.

ex:
---
Removes a random element.

s = {1, 2, 3}
print(s.pop())

Because sets are unordered, the removed item is unpredictable.

ex:
---
Removes everything.

s.clear()
print(s)

Output:
set()

ex:
---
Sets are extremely fast for checking existence.

s = {1, 2, 3}

print(2 in s)
print(10 in s)

Output:

True
False
This is one major reason sets are widely used.

ex:
----
Set Operations (Very Important)

Suppose:

A = {1, 2, 3, 4}
B = {3, 4, 5, 6}
a) Union

All unique elements from both sets.

A∪B

print(A | B)
print(A.union(B))

Output:

{1, 2, 3, 4, 5, 6}
b) Intersection

Common elements.

A∩B

print(A & B)
print(A.intersection(B))

Output:

{3, 4}
c) Difference

Elements in A but not in B.

print(A - B)
print(A.difference(B))

Output:

{1, 2}
d) Symmetric Difference

Elements in either set but not both.

print(A ^ B)
print(A.symmetric_difference(B))

Output:

{1, 2, 5, 6}

subset

print(A.issubset(B))

Output:

True
superset
print(B.issuperset(A))

Output:

True
9. Frozen Set

A frozenset is an immutable version of a set.

fs = frozenset([1, 2, 3])
print(fs)

You cannot modify it:

fs.add(4)

Error occurs.

ex:
---
Set Comprehension

Like list comprehension but with {}.

s = {x*x for x in range(5)}
print(s)

Output:

{0, 1, 4, 9, 16}

Mistakes
========
Mistake 1: Using {} for empty set

Wrong:

s = {}

Correct:

s = set()



ex:
---
Mistake 2: Trying to access by index
s = {1, 2, 3}
print(s[0])

Error:

TypeError

Sets are unordered.


ex:
---
Mistake 3: Adding mutable objects
s.add([1, 2])

Error because lists are mutable.



Q1. Why are sets faster than lists for searching?
Because sets use hash tables, giving average O(1) lookup.

Q2. Can a set contain another set?
No, because sets are mutable.
But it can contain frozenset.

Q3. Why are duplicates removed automatically?
Because set elements are stored as unique hash keys.
